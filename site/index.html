01

Two CLIs, one pane.

Reads ~/.claude/projects/ and ~/.codex/sessions/ into the same view. Every row is tagged with the source CLI so you know which tool ran the conversation.

  • CC Claude Code JSONL
  • CX Codex CLI rollouts

02

Find what you wrote.

Hit Ctrl-F in the preview. Every match across the conversation lights up. The counter ticks as you cycle — ⏎ next, ⇧⏎ previous.

  • Fuzzy & literal modes
  • Highlights persist on cursor move

03

Pick up where you left off.

Cursor on any message, press ⏎ twice. openctx writes a fresh JSONL truncated at the cut, then runs claude --resume (or codex resume) with your edit pre-filled via PTY bracketed paste.

  • Same terminal, or a new window (macOS)
  • Works with Node and Bun via runtime PTY split

02 / Honest answers

Is anything sent off my machine? +

No. openctx is a local-only TUI. It reads the JSONL files Claude Code and Codex CLI already wrote to ~/.claude/projects/ and ~/.codex/sessions/. Nothing is uploaded, indexed remotely, or phoned home.

How does continue conversation work? +

For Claude Code, openctx writes a fresh JSONL beside the original (new UUID, same project) containing the entries up to your cut point, then runs claude --resume <new-uuid>. If the cut is a user message, that text is pre-filled into Claude's input via PTY bracketed paste. For Codex, codex resume <session-id> picks the transcript up whole.

Bun or Node — which do I install under? +

Either. Under Bun, the optional bun-pty is picked up automatically for the resume PTY. Under Node ≥ 20, @lydell/node-pty is used. The runtime is detected at launch — you don't pick.

How do I uninstall? +

npm uninstall -g @12og3r/openctx (or the equivalent for bun / pnpm / yarn). Settings live at ~/.openctx/settings.json; remove it if you want a clean slate.
